Antecedentes –

Fourth Coffee es una cadena global de cafeterías y empresa de café reconocida como una de las marcas de café más influyentes del mundo. La compañía es reconocida por sus bebidas de café especializadas, incluida una amplia gama de bebidas a base de espresso, tés y otras bebidas. Fourth Coffee opera miles de tiendas en todo el mundo.

⸻

Entorno actual –

La empresa está desarrollando aplicaciones nativas en la nube alojadas en Azure.

Sitio web corporativo –
La empresa aloja un sitio web público ubicado en http://www.fourthcoffee.com/.
El sitio web se utiliza para realizar pedidos, así como para ver y actualizar elementos de inventario.

Elementos de inventario –
Además de sus ofertas principales de café, Fourth Coffee amplió recientemente su menú para incluir artículos como almuerzos, snacks y mercancía.
Los miembros del equipo corporativo actualizan constantemente el inventario. Los usuarios pueden personalizar artículos.
Los miembros del equipo corporativo configuran los artículos de inventario y las imágenes asociadas en el sitio web.

Pedidos –
Los asociados en tienda preparan bebidas y artículos personalizados para los clientes.
Los pedidos se realizan en el sitio web para ser recogidos.

⸻

Los componentes de la aplicación procesan los datos de la siguiente manera:
	1.	Azure Traffic Manager enruta la solicitud de pedido del usuario al sitio web corporativo alojado en Azure App Service.
	2.	Azure Content Delivery Network sirve imágenes estáticas y contenido al usuario.
	3.	El usuario inicia sesión en la aplicación a través de un inquilino de Microsoft Entra ID para clientes.
	4.	Los usuarios buscan artículos y realizan pedidos en el sitio web mientras las imágenes se obtienen desde Azure Blob Storage.
	5.	Las personalizaciones de los artículos se colocan en un mensaje de cola de Azure Service Bus.
	6.	Azure Functions procesa las personalizaciones de artículos y guarda los artículos personalizados en Azure Cosmos DB.
	7.	El sitio web guarda los detalles del pedido en Azure SQL Database.
	8.	Los resultados de consultas de SQL Database se almacenan en caché en Azure Cache for Redis para mejorar el rendimiento.

⸻

Requisitos –

La aplicación debe cumplir lo siguiente:
	•	El desarrollo con Azure Cosmos DB debe usar una API nativa que reciba las últimas actualizaciones y almacene datos en formato de documento.
	•	Los costos deben minimizarse para todos los servicios de Azure.
	•	Los desarrolladores deben probar las integraciones de Azure Blob Storage localmente antes de la implementación en Azure. Las pruebas deben ser compatibles con las últimas versiones de las API de Azure Storage.

Sitio web corporativo –
	•	La autenticación y autorización de usuarios debe permitir métodos de inicio de sesión con código de un solo uso y proveedores de identidad social (Google o Facebook).
	•	El contenido web estático debe almacenarse lo más cerca posible de los usuarios finales para reducir la latencia de red.

Elementos de inventario –
	•	Los artículos personalizados leídos desde Azure Cosmos DB deben maximizar el rendimiento mientras garantizan datos precisos para el usuario actual en el sitio web.
	•	El procesamiento de actualizaciones de artículos debe escalar automáticamente y permitir actualizaciones en todo el contenedor de Azure Cosmos DB.
	•	Los artículos de inventario deben procesarse en el orden en que fueron colocados en la cola.
	•	Las imágenes de los artículos deben almacenarse como archivos JPEG en su formato nativo, incluyendo los metadatos EXIF almacenados junto con los datos del blob al cargar el archivo.
	•	La API de artículos de inventario debe acceder de manera segura a los datos de Azure Cosmos DB.

Pedidos –
	•	Los pedidos deben recibir automáticamente los cambios en los artículos de inventario después de que estos se actualicen o se guarden.

⸻

Problemas –
	•	Los desarrolladores están almacenando las credenciales de Azure Cosmos DB en texto claro e inseguro dentro del código de la API de Artículos de Inventario.
	•	El mantenimiento de Azure Cache for Redis en producción ha afectado negativamente el rendimiento de la aplicación.
